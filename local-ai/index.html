Local AI setup

Run an LLM on your own machine via Ollama, then allmeta apps can call it for structured extraction (PICO elements, effect sizes, 2×2 tables from abstracts, GRADE justifications). Everything stays on your device — the model runs locally; the browser just speaks HTTP to 127.0.0.1:11434.

Three-step install

1 — Install Ollama

  • macOS / Windows: download the desktop installer from ollama.com/download.
  • Linux: curl -fsSL https://ollama.com/install.sh | sh

2 — Pull a model

Open a terminal after Ollama is running:

ollama pull llama3.1:8b       # ~4.7 GB, balanced speed + quality
# or
ollama pull qwen2.5-coder:7b  # ~4.7 GB, strong at structured output
# or
ollama pull gpt-oss:20b       # ~13 GB, best reasoning if you have the RAM

Smaller options if you are RAM-limited: llama3.2:3b, gemma2:2b.

3 — Allow this origin (CORS)

By default, Ollama rejects browser requests from other origins. Set OLLAMA_ORIGINS to the exact origin of this page, then restart Ollama.

This page is served from: 

export OLLAMA_ORIGINS='…'

Windows (PowerShell):

$env:OLLAMA_ORIGINS = "…"

Then quit Ollama from the system tray and relaunch it. Click Re-check above.

Privacy model

  • Ollama runs on your computer. The model weights live on your disk; inference happens on your CPU or GPU.
  • This page's JavaScript calls http://127.0.0.1:11434/api/generate. That request is loopback — it never reaches your router, let alone the internet.
  • No telemetry in allmeta. The only external request the hub makes is the initial load (HTML/CSS/JS from GitHub Pages). Browser DevTools → Network tab will confirm.
  • The OLLAMA_ORIGINS setting is the gate — only the origins you list can call your Ollama instance.

Troubleshooting

"not detected" even after install

  • Is Ollama actually running? curl http://localhost:11434/api/tags from a terminal should return JSON.
  • Is OLLAMA_ORIGINS set? Without it, browsers block the request. Check: echo $OLLAMA_ORIGINS (Mac/Linux) or echo %OLLAMA_ORIGINS% (Win).
  • Did you restart Ollama after setting the variable? It only reads env vars at startup.

"Model returned non-JSON"

Some small models ignore the format: "json" hint. Switch to llama3.1:8b or qwen2.5-coder:7b — both respect JSON mode reliably.

Very slow

CPU-only inference for 8B models is ~10 tokens/sec; expect 20–40s for a PICO extraction. If you have a GPU, Ollama uses it automatically on macOS (Metal) and on NVIDIA (CUDA).
